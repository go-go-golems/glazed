    14	# Config Files and Overlays
    15	
    16	Glazed provides first-class support for reading configuration from one or more YAML/JSON files. Files are applied from low → high precedence, every file is recorded as its own parse step, and the result integrates cleanly with environment variables, positional args, and flags.
    17	
    18	- Precedence: Defaults < Config files (low→high) < Env < Positional Args < Flags
    19	- Traceability: Each config file write is logged with `source: config` and metadata such as `{ config_file, index }`, and richer layered flows can also record `{ config_index, config_layer, config_source_name, config_source_kind }`.
    20	- Loading styles: You can load config via simple file middlewares (`FromFile`, `FromFiles`), via resolved layered inputs (`FromResolvedFiles`), or directly from declarative plans (`FromConfigPlan`, `FromConfigPlanBuilder`).
    21	
    22	This guide shows how to load single and multiple files, integrate with Cobra, implement app-level file resolution patterns, use pattern- and custom-mappers, inspect parse steps, and validate config files.
    23	
    24	## Option A: Direct middlewares (library-only)
    25	
    26	Use this approach when you’re embedding Glazed into a service or library and you want explicit, programmatic control over where configuration comes from. You decide the exact order of sources and call the middleware execution yourself. This makes the precedence rules obvious in code and easy to unit test.
    27	
    28	If you already have a declarative `config.Plan`, you can either resolve it yourself and call `FromResolvedFiles(...)`, or load it directly with `FromConfigPlan(...)` / `FromConfigPlanBuilder(...)`.
    29	
    30	```go
    31	package main
    32	
    33	import (
    34	    "github.com/go-go-golems/glazed/pkg/cmds/schema"
    35	    "github.com/go-go-golems/glazed/pkg/cmds/middlewares"
    36	    "github.com/go-go-golems/glazed/pkg/cmds/fields"
    37	    "github.com/spf13/cobra"
    38	)
    39	
    40	func run(cmd *cobra.Command, args []string) error {
    41	    // Define sections
    42	    demo, _ := schema.NewSection(
    43	        "demo", "Demo",
    44	        schema.WithFields(
    45	            fields.New("api-key", fields.TypeString),
    46	            fields.New("threshold", fields.TypeInteger, fields.WithDefault(10)),
    47	        ),
    48	    )
    49	    pls := schema.NewSchema(sections.WithSections(demo))
    50	    parsed := values.New()
    51	
    52	    // Apply middlewares in reverse-precedence order so later sources override earlier ones
    53	    err := sources.Execute(pls, parsed,
    54	        sources.FromDefaults(),                                // Defaults
    55	        sources.FromFiles([]string{"base.yaml"}),  // Config (low → high)
    56	        sources.FromEnv("MYAPP"),                           // Env (prefix)
    57	        sources.FromArgs(args),                            // Positional args
    58	        sources.FromCobra(cmd),                       // Flags
    59	    )
    60	    return err
    61	}
    62	```
    63	
    64	## Option B: Cobra integration (recommended for CLIs)
    65	
    66	If you’re building a CLI, the Cobra integration keeps environment-variable loading and config-file loading separate on purpose. `AppName` turns on the env prefix for the built-in parser path, while `ConfigPlanBuilder` adds explicit config-file discovery. This keeps your command code focused on business logic while Glazed handles the parsing pipeline and debug flags (like `--print-parsed-fields`).
    67	
    68	Use `github.com/go-go-golems/glazed/pkg/cli` to build Cobra commands and attach config processing. The default parser path wires env loading when `AppName` is set; config loading stays explicit through `ConfigPlanBuilder`. If you supply `MiddlewaresFunc`, you replace that built-in chain and must re-add any sources you still want.
    69	
    70	```go
    71	package main
    72	
    73	import (
    74	    "github.com/go-go-golems/glazed/pkg/cli"
    75	    "github.com/go-go-golems/glazed/pkg/cmds"
    76	    "github.com/go-go-golems/glazed/pkg/cmds/schema"
    77	    "github.com/go-go-golems/glazed/pkg/cmds/fields"
    78	)
    79	
    80	func build() (*cobra.Command, error) {
    81	    demo, _ := schema.NewSection(
    82	        "demo", "Demo",
    83	        schema.WithFields(
    84	            fields.New("api-key", fields.TypeString),
    85	            fields.New("threshold", fields.TypeInteger, fields.WithDefault(10)),
    86	        ),
    87	    )
    88	    desc := cmds.NewCommandDescription("demo", cmds.WithSectionsList(demo))
    89	
    90	    // AppName enables env loading in the default parser path (prefix = APPNAME_)
    91	    // ConfigPlanBuilder defines config-file discovery explicitly
    92	    return cli.BuildCobraCommandFromCommand(&DemoBare{desc},
    93	        cli.WithParserConfig(cli.CobraParserConfig{
    94	            AppName: "myapp",
    95	            ConfigPlanBuilder: func(_ *values.Values, _ *cobra.Command, _ []string) (*config.Plan, error) {
    96	                return config.NewPlan(
    97	                    config.WithLayerOrder(config.LayerSystem, config.LayerCWD),
    98	                ).Add(
    99	                    config.ExplicitFile("base.yaml").Named("base").InLayer(config.LayerSystem),
   100	                    config.ExplicitFile("local.yaml").Named("local").InLayer(config.LayerCWD),
   101	                ), nil
   102	            },
   103	        }),
   104	    )
   105	}
   106	```

   108	## Single file
   109	
   110	Load a single YAML/JSON file when your application’s configuration is centralized. The file is parsed into your field sections, and each value update is recorded as a `config` parse step, making it clear where settings came from when debugging.
   111	
   112	```go
   113	package main
   114	
   115	import (
   116	    "github.com/go-go-golems/glazed/pkg/cmds/schema"
   117	    "github.com/go-go-golems/glazed/pkg/cmds/middlewares"
   118	)
   119	
   120	schema_ := schema.NewSchema(/* ... */)
   121	parsed := values.New()
   122	_ = sources.Execute(schema_, parsed,
   123	    sources.FromFile("/etc/myapp/config.yaml"),
   124	)
   125	```
   126	
   127	## Multiple files (overlays)
   128	
   129	Overlays let you compose configuration from multiple files with deterministic precedence. A common pattern is a base file committed to version control plus a local developer override. Glazed applies files in the order you provide (low → high), and the last writer wins. Each file is recorded as its own parse step with `config_file` and `index` metadata so you can audit how a value evolved.
   130	
   131	```go
   132	package main
   133	
   134	import (
   135	    "github.com/go-go-golems/glazed/pkg/cmds/schema"
   136	    "github.com/go-go-golems/glazed/pkg/cmds/middlewares"
   137	)
   138	
   139	files := []string{"base.yaml", "env.yaml", "local.yaml"} // low → high precedence
   140	_ = sources.Execute(schema_, parsed,
   141	    sources.FromFiles(files),
   142	)
   143	```

   145	## Declarative config plans
   146	
   147	If your application needs more than “load these files in this order,” use a declarative config plan from `glazed/pkg/config`. This is the reusable API for expressing layered discovery such as system config, user config, repository-local config, working-directory config, and explicit overrides.
   148	
   149	A plan is built from named source specs and an explicit layer order:
   150	
   151	```go
   152	plan := config.NewPlan(
   153	    config.WithLayerOrder(
   154	        config.LayerSystem,
   155	        config.LayerUser,
   156	        config.LayerRepo,
   157	        config.LayerCWD,
   158	        config.LayerExplicit,
   159	    ),
   160	    config.WithDedupePaths(),
   161	).Add(
   162	    config.SystemAppConfig("myapp").Named("system-app-config"),
   163	    config.XDGAppConfig("myapp").Named("xdg-app-config"),
   164	    config.HomeAppConfig("myapp").Named("home-app-config"),
   165	    config.GitRootFile(".myapp.local.yaml").Named("git-root-local"),
   166	    config.WorkingDirFile(".myapp.local.yaml").Named("cwd-local"),
   167	    config.ExplicitFile(explicitPath).Named("explicit-config-file"),
   168	)
   169	
   170	files, report, err := plan.Resolve(context.Background())
   171	if err != nil {
   172	    return err
   173	}
   174	fmt.Println(report.String())
   175	
   176	err = sources.Execute(
   177	    schema_,
   178	    parsed,
   179	    sources.FromResolvedFiles(files),
   180	    sources.FromDefaults(fields.WithSource(fields.SourceDefaults)),
   181	)
   182	```
   183	
   184	If you do not need the explicit `files, report := plan.Resolve(...)` step, you can load the plan directly:
   185	
   186	```go
   187	err = sources.Execute(
   188	    schema_,
   189	    parsed,
   190	    sources.FromConfigPlan(plan),
   191	    sources.FromDefaults(fields.WithSource(fields.SourceDefaults)),
   192	)
   193	```
   194	
   195	Use the direct plan middlewares when you want Glazed to resolve and load the plan inside the middleware pipeline. Use `FromResolvedFiles(...)` when you want to inspect, test, print, or otherwise reuse the explicit `[]ResolvedConfigFile` output before loading.
   196	
   197	Use this approach when you want both:
   198	
   199	- explicit config source ordering that is easy to review in code
   200	- rich provenance in parsed field history, including `config_layer` and `config_source_name`
   201	
   202	See the dedicated topic [Declarative Config Plans](27-declarative-config-plans.md) and the runnable example `cmd/examples/config-plan` for the full pattern.

   204	## App-level config discovery and patterns
   205	
   206	Many CLIs have a conventional config location (XDG, home dotdir, or `/etc`). Express that policy directly through `config.Plan` so the search order and precedence are visible in code. Pair plan-based discovery with the `--config-file` flag (already provided by the `command-settings` section) so power users can override discovery explicitly.
   207	
   208	Use `github.com/go-go-golems/glazed/pkg/config` to build a plan that discovers a conventional app config plus explicit overrides:
   209	
   210	```go
   211	package main
   212	
   213	import (
   214	    "context"
   215	    "fmt"
   216	    "os"
   217	    "path/filepath"
   218	    "strings"
   219	
   220	    "github.com/go-go-golems/glazed/pkg/cli"
   221	    "github.com/go-go-golems/glazed/pkg/cmds/values"
   222	    glazedconfig "github.com/go-go-golems/glazed/pkg/config"
   223	    "github.com/spf13/cobra"
   224	)
   225	
   226	overrideSibling := func(path string) glazedconfig.SourceSpec {
   227	    return glazedconfig.SourceSpec{
   228	        Name:       "explicit-override",
   229	        Layer:      glazedconfig.LayerExplicit,
   230	        SourceKind: "computed-override-file",
   231	        Optional:   true,
   232	        Discover: func(context.Context) ([]string, error) {
   233	            if strings.TrimSpace(path) == "" {
   234	                return nil, nil
   235	            }
   236	            stem := strings.TrimSuffix(filepath.Base(path), filepath.Ext(path))
   237	            override := filepath.Join(filepath.Dir(path), fmt.Sprintf("%s.override.yaml", stem))
   238	            if _, err := os.Stat(override); err == nil {
   239	                return []string{override}, nil
   240	            } else if os.IsNotExist(err) {
   241	                return nil, nil
   242	            } else {
   243	                return nil, err
   244	            }
   245	        },
   246	    }
   247	}
   248	
   249	builder := func(parsed *values.Values, _ *cobra.Command, _ []string) (*glazedconfig.Plan, error) {
   250	    cs := &cli.CommandSettings{}
   251	    _ = parsed.DecodeSectionInto(cli.CommandSettingsSlug, cs)
   252	
   253	    return glazedconfig.NewPlan(
   254	        glazedconfig.WithLayerOrder(
   255	            glazedconfig.LayerSystem,
   256	            glazedconfig.LayerUser,
   257	            glazedconfig.LayerExplicit,
   258	        ),
   259	    ).Add(
   260	        glazedconfig.SystemAppConfig("myapp").Named("system-app-config"),
   261	        glazedconfig.XDGAppConfig("myapp").Named("xdg-app-config"),
   262	        glazedconfig.HomeAppConfig("myapp").Named("home-app-config"),
   263	        glazedconfig.ExplicitFile(cs.ConfigFile).Named("explicit-config"),
   264	        overrideSibling(cs.ConfigFile),
   265	    ), nil
   266	}
   267	```

   269	## Config file formats and mapping strategies
   270	
   271	Glazed supports both “default-shaped” configs (where the file mirrors your sections and fields) and mappers (which translate arbitrary structures to field updates). Use the default structure for greenfield projects and simple cases—it’s the most transparent. Reach for mappers when you must consume legacy formats, have nested structures that don’t match your field layout, or need to derive multiple fields from one subtree.
   272	
   273	Glazed supports two ways to map config file data into field sections:
   274	
   275	1) Default structure (no mapper): your config matches the section/field shapes directly
   276	
   277	```yaml
   278	# default structure
   279	demo:
   280	  api-key: cfg
   281	  threshold: 33
   282	```
   283	
   284	2) Mappers: use a mapper to transform arbitrary config shapes to section/field assignments.
   285	
   286	### Pattern-based mapper (declarative)
   287	
   288	Pattern mappers describe how to traverse a config tree and map matched values into fields. Patterns support exact segments, wildcards, and named captures (for environment-like keys such as `{env}`). Validation happens both at construction time (syntax, capture references, static targets) and at runtime (required matches, ambiguity, collisions). Prefer named captures over wildcards when you expect multiple values to be collected.
   289	
   290	Use `github.com/go-go-golems/glazed/pkg/cmds/middlewares/patternmapper` to declare mapping rules and pass the mapper to `LoadFieldsFromFile`.
   291	
   292	```go
   293	package main
   294	
   295	import (
   296	    "github.com/go-go-golems/glazed/pkg/cmds/schema"
   297	    "github.com/go-go-golems/glazed/pkg/cmds/middlewares"
   298	    pm "github.com/go-go-golems/glazed/pkg/cmds/middlewares/patternmapper"
   299	    "github.com/go-go-golems/glazed/pkg/cmds/fields"
   300	)
   301	
   302	demo, _ := schema.NewSection("demo", "Demo",
   303	    schema.WithFields(
   304	        fields.New("api-key", fields.TypeString),
   305	        fields.New("dev-api-key", fields.TypeString),
   306	        fields.New("prod-api-key", fields.TypeString),
   307	    ),
   308	)
   309	pls := schema.NewSchema(sections.WithSections(demo))
   310	
   311	mapper, _ := pm.NewConfigMapper(pls,
   312	    pm.MappingRule{
   313	        Source:      "app.{env}.settings",
   314	        TargetSection: "demo",
   315	        Rules: []pm.MappingRule{
   316	            {Source: "api_key", TargetField: "{env}-api-key"},
   317	        },
   318	    },
   319	)
   320	
   321	_ = sources.Execute(pls, values.New(),
   322	    sources.FromFile("config.yaml", middlewares.WithConfigMapper(mapper)),
   323	)
   324	```
   325	
   326	### Custom mapper (Go function)
   327	
   328	Use a custom function when you need full control: conditional logic, array handling, value transformations, or cross-field validation that’s not practical to express with patterns. The function receives the unmarshaled config as `interface{}` and returns a standard `map[sectionSlug]map[paramName]any` for Glazed to apply.
   329	
   330	Provide a `ConfigFileMapper` function to `WithConfigFileMapper` to transform raw config into a `map[sectionSlug]map[paramName]any`:
   331	
   332	```go
   333	package main
   334	
   335	import (
   336	    "github.com/go-go-golems/glazed/pkg/cmds/middlewares"
   337	)
   338	
   339	var mapper middlewares.ConfigFileMapper = func(raw interface{}) (map[string]map[string]interface{}, error) {
   340	    // inspect raw (unmarshaled YAML/JSON) and build the section map
   341	    return map[string]map[string]interface{}{
   342	        "demo": {"api-key": "secret", "threshold": 5},
   343	    }, nil
   344	}
   345	
   346	_ = sources.Execute(pls, values.New(),
   347	    sources.FromFile("config.yaml", middlewares.WithConfigFileMapper(mapper)),
   348	)
   349	```

   351	## Inspecting parse steps (`--print-parsed-fields`)
   352	
   353	Parsing is not a black box—every write records its source and any relevant metadata. Enable `--print-parsed-fields` to see the exact sequence of updates for each field. This is invaluable when debugging precedence issues (for example, “why didn’t my local override win?”) or auditing where a value originated.
   354	
   355	Add the `command-settings` section (done automatically by the Cobra parser unless disabled) and run with `--print-parsed-fields` to see where a value came from:
   356	
   357	```yaml
   358	demo:
   359	  api-key:
   360	    log:
   361	      - source: config
   362	        metadata: { config_file: base.yaml, index: 0 }
   363	        value: base
   364	      - source: config
   365	        metadata: { config_file: local.yaml, index: 1 }
   366	        value: local
   367	    value: local
   368	```

   370	## Validation
   371	
   372	Validate configs early to catch mistakes before runtime. For default-shaped files, check for unknown sections/fields and type errors. For pattern-based configs, instantiate a mapper and call `Map` in a validate-only pass; the mapper will fail fast on missing required matches, ambiguous patterns, or invalid targets. These validators are small enough to run in CI and provide crisp error messages for contributors.
   373	
   374	You can validate config files before applying them.
   375	
   376	### Default-structured validator (unknown keys + type checks)
   377	
   378	Apply this validator to YAML/JSON files that mirror your sections. It’s conservative by design: any unexpected section or field is flagged, and values are type-checked against your field definitions. This keeps configs tidy and prevents silent drift as fields evolve.
   379	
   380	```go
   381	package main
   382	
   383	import (
   384	    "fmt"
   385	    "os"
   386	    "strings"
   387	
   388	    "github.com/go-go-golems/glazed/pkg/cmds/schema"
   389	    "github.com/go-go-golems/glazed/pkg/cmds/fields"
   390	    "gopkg.in/yaml.v3"
   391	)
   392	
   393	func validateConfigFile(schema_ *schema.Schema, path string) error {
   394	    b, err := os.ReadFile(path)
   395	    if err != nil { return err }
   396	    var raw map[string]interface{}
   397	    if err := yaml.Unmarshal(b, &raw); err != nil { return err }
   398	
   399	    issues := []string{}
   400	    for sectionSlug, v := range raw {
   401	        section, ok := schema_.Get(sectionSlug)
   402	        if !ok { issues = append(issues, fmt.Sprintf("unknown section %s", sectionSlug)); continue }
   403	        kv, ok := v.(map[string]interface{})
   404	        if !ok { issues = append(issues, fmt.Sprintf("section %s must be an object", sectionSlug)); continue }
   405	        pds := section.GetDefinitions()
   406	        known := map[string]bool{}
   407	        pds.ForEach(func(pd *fields.Definition) { known[pd.Name] = true })
   408	        for key, val := range kv {
   409	            if !known[key] { issues = append(issues, fmt.Sprintf("unknown field %s.%s", sectionSlug, key)); continue }
   410	            pd, _ := pds.Get(key)
   411	            if _, err := pd.CheckValueValidity(val); err != nil {
   412	                issues = append(issues, fmt.Sprintf("invalid value for %s.%s: %v", sectionSlug, key, err))
   413	            }
   414	        }
   415	    }
   416	    if len(issues) > 0 { return fmt.Errorf(strings.Join(issues, "\n")) }
   417	    return nil
   418	}
   419	```
   420	
   421	Validate overlays per-file or implement overlay-aware required semantics if needed.
   422	
   423	### Pattern-based validator (`mapper.Map`)
   424	
   425	For declarative mappings, the mapper is your validator. Build it once per app (construction validates static aspects) and call `Map` on the raw config (runtime semantics validate dynamic aspects). Error messages include path hints and prefix-aware field names to accelerate debugging.
   426	
   427	```go
   428	package main
   429	
   430	import (
   431	    "os"
   432	    "github.com/go-go-golems/glazed/pkg/cmds/schema"
   433	    pm "github.com/go-go-golems/glazed/pkg/cmds/middlewares/patternmapper"
   434	    "gopkg.in/yaml.v3"
   435	)
   436	
   437	rules, _ := pm.LoadRulesFromFile("mappings.yaml")
   438	mapper, _ := pm.NewConfigMapper(pls, rules...)
   439	b, _ := os.ReadFile("config.yaml")
   440	var raw map[string]interface{}
   441	_ = yaml.Unmarshal(b, &raw)
   442	if _, err := mapper.Map(raw); err != nil {
   443	    // invalid config (required missing, collisions, unknown target params, ...)
   444	}
   445	```

   447	## Tips & best practices
   448	
   449	These guidelines help keep your configuration predictable across environments and easy to reason about during reviews and incident response.
   450	
   451	- Keep overlays small and ordered: `base.yaml`, `env.yaml`, `local.yaml`.
   452	- Prefer named captures over wildcards in pattern rules when collecting multiple values.
   453	- Use `AppName` in `CobraParserConfig` to enable env overrides automatically on the built-in parser path.
   454	- Record parse sources with `sources.WithSource("config")` (done for you by the config middlewares).

   474
